MaterialDesignInXAML

設計師 & 工程師使用手冊 — Allring Edition v1

1. 系統概覽

1.1 什麼是 MaterialDesignInXAML

MaterialDesignInXAML（簡稱 MDIXAML）是一套開源的 WPF UI 元件庫，將 Google Material Design 的設計語言完整移植到 Windows Presentation Foundation（WPF）平台。

它提供了：

  • 70+ 個已樣式化的 WPF 控件（按鈕、卡片、導航、對話框等）
  • 完整的色彩系統（19 種 Material 調色盤 + 語義筆刷）
  • 字型系統（內建 Roboto / Noto 字型 + Type Scale）
  • 主題引擎（Light/Dark 模式、動態配色、WCAG 無障礙對比度）
  • MD3 支援（Material Design 3 動態配色方案，從種子色自動展開）
💡 與 Google Material Design 的關係
MDIXAML 遵循 Google Material Design 規範，但針對 WPF 平台做了適配。它並非 Google 官方產品，而是社群維護的開源專案（MIT 授權）。目前同時支援 Material Design 2 和 Material Design 3。

2. 色彩系統

2.1 三層架構

MDIXAML 的色彩系統分為三個層次，從底層到頂層：

Layer 1：基底色（Base Theme Colors）

定義在 BaseThemeColors.cs 和 MaterialDesignTheme.BaseThemeColors.xaml

提供最基礎的中性色階，不帶任何品牌色彩：

類別色階範圍用途
Neutral50 ~ 900（灰階）背景、邊框、分隔線
White0 ~ 1000（透明度梯度）Light 模式的基底
Black0 ~ 1000（透明度梯度）Dark 模式的基底

Layer 2：色票（Swatch / Color Palette）

定義在 MaterialDesignColors.Wpf/Themes/

包含 19 種 Material 調色盤，每種有完整的色階：

每種色票包含 Primary 50-900（10 級）+ Secondary 100-700（4 級），每級都有對應的前景色（Foreground）確保文字可讀性。

Layer 3：語義筆刷（Semantic Brush）

定義在 MaterialDesignTheme.Light.xaml / MaterialDesignTheme.Dark.xaml

這是元件實際引用的層級。約 67 個語義筆刷，例如：

  • MaterialDesign.Brush.Primary — 主要品牌色
  • MaterialDesign.Brush.Primary.Light — 主色淺色變體
  • MaterialDesign.Brush.Primary.Dark — 主色深色變體
  • MaterialDesign.Brush.Primary.Foreground — 主色上的前景文字
  • MaterialDesign.Brush.Secondary — 次要品牌色
  • MaterialDesign.Brush.Background — 頁面背景
  • MaterialDesign.Brush.Card.Background — 卡片背景
  • MaterialDesign.Brush.Body — 內文文字
  • ...等共 67 個
💡 設計師重點
你在 Figma 中定義的顏色 Token，最終會對應到 Layer 3 的語義筆刷。例如你的 color/primary = MDIXAML 的 MaterialDesign.Brush.Primary。

⚠️ 種子色展開機制
MDIXAML 使用 HCT（Hue, Chroma, Tone）色彩空間，從你的種子色提取色相和彩度，然後在不同明度（Tone）上生成完整的色調盤（Tonal Palette）。這意味著自動展開的色階可能與萬潤 token 中手動定義的色階略有差異。方案 A 接受此差異以獲得最快的整合速度。

2.3 萬潤品牌色整合

🎨 萬潤品牌色 → MDIXAML 對應

萬潤 Token色碼MDIXAML 用途
--color-primary #016194 作為 CustomColorTheme.PrimaryColor 種子色
--color-secondary #F48001 作為 CustomColorTheme.SecondaryColor 種子色

快速套用（App.xaml）

XAML 
<Application.Resources>
  <ResourceDictionary>
    <ResourceDictionary.MergedDictionaries>

      <!-- 萬潤品牌色主題 -->
      <materialDesign:CustomColorTheme
        BaseTheme="Light"
        PrimaryColor="#016194"
        SecondaryColor="#F48001" />

      <!-- MD3 元件樣式 -->
      <ResourceDictionary
        Source="pack://application:,,,/MaterialDesignThemes.Wpf;component/Themes/MaterialDesign3.Defaults.xaml" />

    </ResourceDictionary.MergedDictionaries>
  </ResourceDictionary>
</Application.Resources>

💡 只需這兩個色碼，MDIXAML 會自動展開為完整的 MD3 配色方案（29 個 Color Role + Light/Dark 變體）。

2.6 色彩無障礙（Accessibility）

MDIXAML 內建 ColorAdjustment 機制，自動確保 WCAG 對比度：

屬性預設值說明
DesiredContrastRatio4.5WCAG AA 標準（一般文字）
ContrastMediumLow / Medium / High 三種對比度等級
ColorsAll對所有色彩套用對比度調整
XAML 
<materialDesign:CustomColorTheme BaseTheme="Light"
  PrimaryColor="#016194"
  SecondaryColor="#F48001">
  <materialDesign:CustomColorTheme.ColorAdjustment>
    <materialDesign:ColorAdjustment
      DesiredContrastRatio="4.5"
      Contrast="Medium"
      Colors="All" />
  </materialDesign:CustomColorTheme.ColorAdjustment>
</materialDesign:CustomColorTheme>

3.2 中文字型注意事項

⚠️ 中文字型相容性

MDIXAML 內建的 Roboto（MD2）和 Noto（MD3）主要涵蓋拉丁字母。對於繁體中文內容：

  • Noto Sans TC（MD3 的 Noto 家族有中文版）— 推薦作為中文字型
  • Microsoft JhengHei（微軟正黑體）— Windows 內建，最安全的後備字型
  • 建議的 FontFamily 設定：Noto Sans TC, Microsoft JhengHei, sans-serif

替換字型的方法

XAML 
<!-- 在 App.xaml 中覆寫 MaterialDesignFont -->
<FontFamily x:Key="MaterialDesignFont">
  pack://application:,,,/YourApp;component/Fonts/#Noto Sans TC
</FontFamily>

或在 .csproj 設定 <IncludeMaterialDesignFont>False</IncludeMaterialDesignFont> 停用內建字型嵌入。

5. 客製化指南

5.1 四種客製化層級

1
BundledTheme
從預設調色盤選色
最快上手，5 分鐘完成
2
CustomColorTheme ⭐
用任意 Hex 色碼
萬潤推薦方案
3
PaletteHelper
Runtime 動態切換
適合需要換色場景
4
完全自定
自定色票 + 元件覆寫
最大彈性，最多工作量

Level 1: BundledTheme — 預設調色盤選色

XAML 
<materialDesign:BundledTheme
  BaseTheme="Light"
  PrimaryColor="Teal"
  SecondaryColor="Amber" />

可用的 PrimaryColor 值：Red, Pink, Purple, DeepPurple, Indigo, Blue, LightBlue, Cyan, Teal, Green, LightGreen, Lime, Yellow, Amber, Orange, DeepOrange, Brown, Grey, BlueGrey

Level 2: CustomColorTheme — 萬潤推薦 ⭐

XAML — 萬潤品牌色配置 
<materialDesign:CustomColorTheme
  BaseTheme="Light"
  PrimaryColor="#016194"
  SecondaryColor="#F48001" />

Level 3: PaletteHelper — Runtime 動態切換

C# 
var paletteHelper = new PaletteHelper();
var theme = paletteHelper.GetTheme();

// 切換品牌色
theme.SetPrimaryColor((Color)ColorConverter.ConvertFromString("#016194"));
theme.SetSecondaryColor((Color)ColorConverter.ConvertFromString("#F48001"));

// 切換 Light / Dark
theme.SetBaseTheme(BaseTheme.Dark);

paletteHelper.SetTheme(theme);

Level 4: 完全自定

包含：自定色票 XAML、覆寫特定元件 ResourceDictionary、自訂 Assist 屬性預設值。詳見 6.4 進階客製化。

6.3 主題切換（Runtime）

PaletteHelper 完整 API

C# 
using MaterialDesignThemes.Wpf;
using System.Windows.Media;

// 取得 PaletteHelper 實例
var paletteHelper = new PaletteHelper();

// 取得當前主題
var theme = paletteHelper.GetTheme();

// === 修改品牌色 ===
theme.SetPrimaryColor((Color)ColorConverter.ConvertFromString("#016194"));
theme.SetSecondaryColor((Color)ColorConverter.ConvertFromString("#F48001"));

// === 切換 Light / Dark ===
theme.SetBaseTheme(BaseTheme.Light);  // 或 BaseTheme.Dark

// === 套用主題 ===
paletteHelper.SetTheme(theme);

// === 監聽主題變更 ===
var themeManager = paletteHelper.GetThemeManager();
if (themeManager != null)
{
    themeManager.ThemeChanged += (s, e) =>
    {
        // 主題已變更，更新 UI
        Debug.WriteLine($"New primary: {e.NewTheme.PrimaryMid.Color}");
    };
}

7.2 常見問題 FAQ

Q: MDIXAML 支援哪些 .NET 版本？ ▶
支援 .NET Framework 4.6.2+、.NET 8.0（Windows）、.NET 10.0（Windows）。
Q: 如何同時支援 Light 和 Dark 模式？ ▶
使用 PaletteHelper 的 SetBaseTheme() 方法在 Runtime 切換。也可以設定 BaseTheme="Inherit" 讓應用程式跟隨 Windows 系統主題。
Q: CustomColorTheme 和 BundledTheme 差在哪？ ▶
BundledTheme 只能從 19 種預設調色盤選色。CustomColorTheme 接受任意 Hex 色碼，更適合品牌客製化。兩者的主題引擎一樣，只是色彩來源不同。
Q: 種子色自動展開的色階會不會不好看？ ▶
MD3 的 HCT 演算法經過大量測試，展開的色階在絕大多數情況下都是和諧的。如果對特定色階不滿意，可以用 Level 4 客製化方式手動覆寫個別 Brush。
Q: 如何在 Figma 中與 MDIXAML 對齊？ ▶
建議在 Figma 中使用 Material Design 3 的 Design Kit，並建立 Variable 命名對照表（見本手冊 2.3 節）。關鍵是讓 Figma 的 Token 名稱與 XAML 的 Resource Key 可以對應。
